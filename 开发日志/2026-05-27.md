## 本日目标

今天目标是启动卡咔桌宠方向，先完成桌面身体原型，而不是直接接复杂对话系统。

本日选择的桌宠技术组合：

```text
Tauri 2 + React/Vite + TypeScript + PixiJS
```

设计判断：

- Tauri 负责透明桌面窗口、置顶、无边框和原生窗口位置。

- 当前先不接 `kaka-core`，先把桌宠身体、状态和基础互动做好。

## 关键修正

### 右键菜单裁切

最初右键菜单高度接近 `280x280` 窗口高度，底部“退出”会被窗口裁掉。

处理方式：

- 缩小菜单宽度、行高和间距。
- 根据窗口尺寸限制菜单坐标。
- 菜单保持两列，作为当前开发调试入口。

### 右键菜单遮挡卡咔

当前右键菜单会遮挡卡咔本体，这是现阶段可接受的问题。

原因：

- 窗口只有 `280x280`。
- 菜单必须画在透明窗口内部。
- 如果直接放大透明窗口，会增加桌面误点区域。

后续正式版更适合改成：

- 系统托盘菜单。
- 单独设置窗口。
- 更轻量的右键控制入口。

### 拖拽状态恢复

第一版使用 Tauri 原生 `startDragging()`，Web 侧有时收不到拖拽结束事件，导致拖拽状态恢复不稳定。

后续改为自定义拖拽：

- 读取光标位置。
- 读取窗口位置。
- 根据光标位移设置窗口位置。
- 拖拽状态由 React 自己控制，松手后恢复拖拽前状态。

Tauri 权限相应改为：

```text
core:window:allow-cursor-position
core:window:allow-outer-position
core:window:allow-set-position
```

### 连接测试

用户提出暂时没有必要让桌宠直接调用大模型，但希望确认桌宠能连接核心大脑。

本轮新增一个轻量连接测试：

```text
右键菜单 -> 连接测试
-> Tauri 命令 check_kaka_core_health
-> TCP 请求 http://127.0.0.1:8001/health
-> 成功显示核心信号正常
-> 失败显示信号弱
```

实现判断：

```text
只检查 /health，不走 /v1/chat。
这样能验证桌宠和 kaka-core 的本地连接，又不引入大模型调用和聊天输入框。
```

### 窗口位置记忆

为了让桌宠更像常驻桌面小工具，本轮新增窗口位置记忆：

```text
拖拽结束 -> 读取窗口位置 -> localStorage 保存坐标
下次启动 -> 读取 localStorage -> setPosition 恢复坐标
```

没有新增数据库，也没有改 `kaka-core`。

### 随机待机表现

本轮新增低打扰随机待机：

```text
仅在待机状态触发
随机 45-120 秒一次
随机冒一句短气泡，或短暂切到开心 / 思考 / 困困
用户操作会打断随机动作
```

设计判断：

```text
不接大模型也能增强“活着”的感觉
频率不能太高，避免烦人
随机动作不应覆盖用户手动选择的状态
```

### 菜单整理和重置位置

随着功能增多，右键菜单开始承担连接测试、状态切换、退出等入口。

本轮做了轻量分组：

```text
顶部：连接测试 / 重置位置
中间：切换状态
底部：退出
```

`重置位置` 使用 Tauri `center()`，并清除保存的窗口坐标。对应新增权限：

```text
core:window:allow-center
```

## 下一步建议

下一步有两条路线。

体验优先：

```text
系统托盘
显示/隐藏
开机自启
设置窗口
```

大脑接入：

```text
桌宠固定输入或小输入框
-> 组装 platform=desktop 的 MessageEvent
-> POST http://127.0.0.1:8001/v1/chat
-> 把 KakaResponse 文本显示到气泡
-> 请求中显示加载中
-> 收到回复显示收到消息或开心
-> 请求失败显示信号弱
```

当前判断：如果暂时没有明确桌宠对话用途，不急着接大模型；优先继续完善桌宠作为常驻小工具的体验。
