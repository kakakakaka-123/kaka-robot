# 2026-05-27 开发日志

## 本日目标

今天目标是启动卡咔桌宠方向，先完成桌面身体原型，而不是直接接复杂对话系统。

本日选择的桌宠技术组合：

```text
Tauri 2 + React/Vite + TypeScript + PixiJS
```

设计判断：

- Tauri 负责透明桌面窗口、置顶、无边框和原生窗口位置。
- Tauri 负责系统托盘入口、显示/隐藏和退出等原生常驻能力。
- React / TypeScript 负责交互状态、菜单和气泡。
- PixiJS 负责透明 PNG 贴图渲染和轻量状态动作。
- 当前先不接 `kaka-core`，先把桌宠身体、状态和基础互动做好。

## 关键修正

### 右键菜单裁切

最初右键菜单高度接近 `280x280` 窗口高度，底部“退出”会被窗口裁掉。

处理方式：

- 缩小菜单宽度、行高和间距。
- 根据窗口尺寸限制菜单坐标。
- 菜单保持两列，作为当前开发调试入口。

### 右键菜单遮挡卡咔

当前右键菜单会遮挡卡咔本体，这是现阶段可接受的问题。

原因：

- 窗口只有 `280x280`。
- 菜单必须画在透明窗口内部。
- 如果直接放大透明窗口，会增加桌面误点区域。

后续正式版更适合改成：

- 系统托盘菜单。
- 单独设置窗口。
- 更轻量的右键控制入口。

### 拖拽状态恢复

第一版使用 Tauri 原生 `startDragging()`，Web 侧有时收不到拖拽结束事件，导致拖拽状态恢复不稳定。

后续改为自定义拖拽：

- 读取光标位置。
- 读取窗口位置。
- 根据光标位移设置窗口位置。
- 拖拽状态由 React 自己控制，松手后恢复拖拽前状态。

Tauri 权限相应改为：

```text
core:window:allow-cursor-position
core:window:allow-outer-position
core:window:allow-set-position
```

### 连接测试

用户提出暂时没有必要让桌宠直接调用大模型，但希望确认桌宠能连接核心大脑。

本轮新增一个轻量连接测试：

```text
右键菜单 -> 连接测试
-> Tauri 命令 check_kaka_core_health
-> TCP 请求 http://127.0.0.1:8001/health
-> 成功显示核心信号正常
-> 失败显示信号弱
```

实现判断：

```text
只检查 /health，不走 /v1/chat。
这样能验证桌宠和 kaka-core 的本地连接，又不引入大模型调用和聊天输入框。
```

### 窗口位置记忆

为了让桌宠更像常驻桌面小工具，本轮新增窗口位置记忆：

```text
拖拽结束 -> 读取窗口位置 -> localStorage 保存坐标
下次启动 -> 读取 localStorage -> setPosition 恢复坐标
```

没有新增数据库，也没有改 `kaka-core`。

### 随机待机表现

本轮新增低打扰随机待机：

```text
仅在待机状态触发
随机 45-120 秒一次
随机冒一句短气泡，或短暂切到开心 / 思考 / 困困
用户操作会打断随机动作
```

设计判断：

```text
不接大模型也能增强“活着”的感觉
频率不能太高，避免烦人
随机动作不应覆盖用户手动选择的状态
```

### 菜单整理和重置位置

随着功能增多，右键菜单开始承担连接测试、状态切换、退出等入口。

本轮做了轻量分组：

```text
顶部：连接测试 / 重置位置
中间：切换状态
底部：退出
```

`重置位置` 使用 Tauri `center()`，并清除保存的窗口坐标。对应新增权限：

```text
core:window:allow-center
```

### 系统托盘基础版

桌宠作为常驻小工具，需要一个不依赖桌宠本体右键菜单的稳定入口。本轮新增 Tauri 系统托盘：

```text
托盘图标
-> 显示/隐藏卡咔
-> 重置位置
-> 连接测试
-> 退出
```

实现判断：

```text
显示/隐藏和退出由 Rust/Tauri 侧直接处理。
重置位置和连接测试由托盘菜单向前端发事件，再复用已有前端逻辑。
这样托盘重置位置时也会清理 localStorage，不会下次启动又回到旧坐标。
```

Tauri 依赖新增 feature：

```text
tray-icon
image-ico
```

### 开机自启基础版

桌宠已经从任务栏占位切到托盘常驻，下一步补上开机自启，让它更像真正常驻桌宠。

本轮新增：

```text
tauri-plugin-autostart
托盘菜单 -> 开机自启
```

实现判断：

```text
托盘菜单在 Rust/Tauri 侧，所以直接使用插件的 Rust API。
开机自启菜单项使用 CheckMenuItem，勾选状态来自 autolaunch().is_enabled()。
点击菜单后执行 enable() 或 disable()，成功后同步勾选状态。
```

### 设置窗口基础版

托盘菜单继续增加功能会变乱，因此本轮新增一个小型设置窗口。

实现方式：

```text
同一个 Vite/React 包
主窗口：默认渲染 App
设置窗口：index.html?view=settings 渲染 SettingsApp
```

设置项保存：

```text
localStorage key: kaka.desktopPet.settings
```

当前设置内容：

```text
idleAmbientEnabled: 是否启用随机待机气泡/小动作
idleSleepDelayMs: 1 分钟 / 2 分钟 / 5 分钟 / 不自动睡觉
```

跨窗口同步：

```text
设置窗口写入 localStorage
-> emit kaka-settings-updated
-> 主桌宠窗口重新读取设置
-> 重新安排随机待机和闲置睡觉计时器
```

```text
启动参数区分正常启动 / 开机自启启动

```

```text
桌宠固定输入或小输入框
```
