# 2026-05-05 开发日志

## 今日重点

今天主要围绕 Web 管理台的正式记忆管理做收口。

目标不是扩大后台范围，而是把用户实际会用的正式记忆页、提示预演和对话复盘补完整：

```text
最新记忆优先显示
每页 50 条分页
正式记忆新增
正式记忆编辑
提示预演能看到真实回复上下文
对话复盘能回查真实回复记录
```

最近对话、输入分析和候选区仍保留在管理 API、脚本和数据库中，不再作为日常 Web 页面暴露。

## 正式记忆分页和排序

正式记忆列表后端最终改为按 `MemoryRecord.id.desc()` 排序，让最新写入的记忆优先展示。归档、恢复、新增和删除后会刷新分页，避免因为更新时间变化或状态切换导致编号看起来乱跳。

`/admin/api/memories` 现在支持：

```text
limit
offset
total
```

前端默认每页 50 条，显示当前页、总页数、当前条目范围和总数，并提供上一页/下一页。

## 提示预演

新增管理接口：

```text
POST /admin/api/reply-context/preview
```

它复用真实回复上下文组装逻辑，前端在“提示预演”页展示：

```text
System Prompt
User Prompt
metadata
used_memory_ids
memory_count
memory_injection_enabled
```

这个功能主要用于调试 prompt 和长期记忆注入，确认一条新消息在真实回复前到底会给模型什么上下文。

## 对话复盘

新增对话复盘页，用于回查卡咔已经回复过的真实对话，不再展示普通观察记录。

后端能力：

```text
GET /admin/api/conversations
GET /admin/api/conversations/{input_id}
```

列表接口支持 `limit / offset / total`，前端固定每页 50 条。详情接口会读取历史 `outputs.metadata`，按 `used_memory_ids` 和 `short_context_input_ids` 反查当次回复命中的正式记忆和短期上下文。

前端布局最终确定为：

```text
左侧：卡咔回复列表
右侧：对话详情和回复证据
详情下方：短期上下文在左，命中的正式记忆在右
短期上下文和记忆都使用固定高度滚动窗口
```

筛选条件移除了“输出来源”和“输出原因”，因为对话复盘只服务于真实回复回查。

## 自动任务运行状态

运行状态页补充了自动候选分析和自动候选区复核的手动触发按钮。

后端接口：

```text
POST /admin/api/auto-jobs/{job_name}/trigger
```

手动触发会传 `force=true`，绕过 `not_analyzed >= 50` 和 `pending >= 20` 的数量门槛。自动任务仍然会写入 `auto_job_runs`，metadata 里记录 `force`。

同时补了跨进程自动任务锁：

```text
auto_job:auto_analysis
auto_job:auto_review
```

如果本机还有旧的 `kaka-core/run.py` 进程在运行，到整点仍会继续检查自动任务；没有任何 `kaka-core` 进程运行时，自动任务不会自己执行。多个进程同时触发时，只有拿到锁的进程会执行，拿不到锁会写入 skipped 记录。

## 当前结论

正式记忆管理现在已经补齐基础 CRUD 中缺的“增”和“改”，再加上已有的归档、恢复、硬删除，Web 端已经能覆盖日常正式记忆维护。

## 短期上下文第一版

随后补上回复前短期上下文。

新增配置：

```text
SHORT_CONTEXT_ENABLED=true
SHORT_CONTEXT_LIMIT=20
SHORT_CONTEXT_MAX_CHARS=1200
SHORT_CONTEXT_WINDOW_MINUTES=30
```

第一版规则：

```text
只读 inputs / outputs，不建新表
按同一 platform / scene_type / scene_id 取最近输入
排除当前 event_id
只看最近 30 分钟内的记录
默认最多 20 条输入记录
总字符上限 1200
普通观察消息和已有卡咔回复都可以进入上下文
```

上下文接入 `kaka_core.context.builder`，和长期记忆一起组成回复前 prompt。metadata 会记录：

```text
short_context_enabled
short_context_count
short_context_input_ids
short_context_limit
short_context_max_chars
```

本轮补充测试：

```text
同场景最近消息能进入 User Prompt
不同场景不会串
当前消息不会重复进入短期上下文
关闭 SHORT_CONTEXT_ENABLED 后不注入
/admin 回复上下文预览能看到短期上下文 metadata
```
