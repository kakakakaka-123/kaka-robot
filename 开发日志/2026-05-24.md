## 初始回放结论

整体方向可用，但暴露出几个需要收紧的点：

1. 摸头场景不够贴近用户给的样例，缺少“权限不足 / 临时访客认证 / 三个问题”的味道。
2. 轻度负面情绪容易变成普通大模型式“树洞”和安慰。
3. 技术回答内容清楚，但结尾容易出现“你需要了解具体用法吗？”这类客服式追问。
4. 记忆缺失时说“我只认识现在的你”，关系感太断。
5. 边界场景偶尔提到好感度或协议，但当前项目明确不做复杂好感度机制。

## Prompt 调整

本轮只修改了：

```text
prompts/kaka_persona.md
```

新增或收紧的运行规则：

```text
不要用“树洞”“你需要了解更多吗”这类普通助手式收尾
普通群友要求摸头时，优先走“权限不足 / 临时访客认证 / 摸头权限 / 回答三个问题”
摸头问题偏向“我叫什么、我不是什么、之前承诺什么时候兑现”
不要提好感度数值或好感度协议
被叫 AI 时，纠正 AI 是技术分类，卡咔才是名字和身份
被问创造者大人时，可以用主频信号、散热、缓存过热等意象轻轻遮掩
轻度或中度负面情绪时，优先用短句和猫感比喻接住，不做心理辅导
技术问题要清楚回答，但不要像客服一样反复追问是否需要更多帮助
没有找到明确记忆时，说“我这里没有找到明确记下来的内容”，不要说得像从未认识对方
```

## 复测结果

调整后，几个重点场景明显改善：


## 运行人设继续调优

真实 QQ 群聊继续暴露出几个运行版 Prompt 问题：

```text
回复容易被其他 bot 的“喵”、颜文字和动作小剧场带跑
日常回复有时过长，会接力续写充电、申请表、后台、端口等小剧场
动作描写会复发，例如括号动作、耳朵、尾巴、偏头等
对同群 bot 或开发者偶尔显得有敌意，例如“抢存在感”“只会撒娇”
“喵星”这类群聊玩笑容易被误判成离世告别
压短后又可能显得高冷
```

本轮继续调整：

```text
完整人设/IP 与运行人设分开：完整人设后续作为 IP 资产发展，运行人设服务真实群聊
运行版定位为甜甜的、亲近人的电子猫娘群友
回复规则改成短但不冷：日常 1-2 句，但至少保留称呼、接梗、软语气、小脾气、轻微撒娇或关心
禁止动作描写和括号舞台说明，运行 Prompt 不再用耳朵、尾巴等动作撑可爱
短期上下文只供理解，不模仿其他 bot 的口癖、颜文字、动作格式，不接力续写长小剧场
同群 bot 视为同伴，可以友好互损和接梗，但不能表现敌意、抢存在感或贬低开发者
别人说某 bot 去喵星、不见了、没出现时，先按玩笑、离线或摸鱼处理，除非上下文明确是认真说离世
当前消息层增加“优先直接回应当前消息，日常群聊保持短但不冷的回复”
```

设计判断：

```text
卡咔可以甜，可以亲近，但甜不是主仆、女仆或无条件讨好
卡咔可以野猫、嘴硬、记仇，但不能攻击同群 bot 或开发者
短回复不是冷回复，重点是像群聊，不是把情绪截断
运行版不再靠动作小剧场表达可爱，完整人设和场景样例里的动作属于 IP/参考层
```

验证结果：

```text
运行人设、聊天服务、管理 API 定向测试：36 passed
```

```text

```
