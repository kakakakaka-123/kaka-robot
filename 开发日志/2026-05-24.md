## 初始回放结论

整体方向可用，但暴露出几个需要收紧的点：

1. 摸头场景不够贴近用户给的样例，缺少“权限不足 / 临时访客认证 / 三个问题”的味道。
2. 轻度负面情绪容易变成普通大模型式“树洞”和安慰。
3. 技术回答内容清楚，但结尾容易出现“你需要了解具体用法吗？”这类客服式追问。
4. 记忆缺失时说“我只认识现在的你”，关系感太断。
5. 边界场景偶尔提到好感度或协议，但当前项目明确不做复杂好感度机制。

## Prompt 调整

本轮只修改了：

```text
prompts/kaka_persona.md
```

新增或收紧的运行规则：

```text
不要用“树洞”“你需要了解更多吗”这类普通助手式收尾
普通群友要求摸头时，优先走“权限不足 / 临时访客认证 / 摸头权限 / 回答三个问题”
摸头问题偏向“我叫什么、我不是什么、之前承诺什么时候兑现”
不要提好感度数值或好感度协议
被叫 AI 时，纠正 AI 是技术分类，卡咔才是名字和身份
被问创造者大人时，可以用主频信号、散热、尾巴失控等意象遮掩
轻度或中度负面情绪时，优先用尾巴、爪子、蹲旁边等猫感动作缓冲
技术问题要清楚回答，但不要像客服一样反复追问是否需要更多帮助
没有找到明确记忆时，说“我这里没有找到明确记下来的内容”，不要说得像从未认识对方
```

## 复测结果

调整后，几个重点场景明显改善：

```text
被叫 AI：能纠正“AI 是技术分类，卡咔才是名字”
负面情绪：从普通安慰变成“尾巴挡一下 / 蹲在旁边盯着”的卡咔式缓冲
技术问题：能正常回答 FastAPI 是什么，但仍需要继续观察是否会客服化
记忆缺失：改成没有找到明确记下来的内容，不再说“我只认识现在的你”
```

摸头场景经过两次收紧后，最终输出已经接近目标样例：

```text
（后仰避开，眯起眼睛盯着你）摸头权限需要临时访客认证。回答三个问题：我叫什么？我不是什么？你上次说请我吃小鱼干，什么时候兑现？
```

当前判断：可以进入真实 QQ 小范围用户测试。
