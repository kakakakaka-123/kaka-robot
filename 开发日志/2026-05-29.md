## 桌宠轻量输入入口和最小核心对话接入

本轮完成桌宠对话入口的第二步和第三步：先做输入入口骨架，再接入 `kaka-core /v1/chat` 最小链路。

新增能力：

```text
右键菜单新增“说话”入口。
点击后打开小输入框，支持发送、取消和 Esc 关闭。
输入框限制 80 字，打开或发送中时不会被随机待机和闲置睡觉打断。
发送时切到 loading 状态，并显示“卡咔思考中...”。
Rust 后端组装 platform=desktop 的 MessageEvent，请求 http://127.0.0.1:8001/v1/chat。
从 KakaResponse.actions 中读取第一条 send_text 文本，并显示到桌宠气泡。
请求失败时切到 weakSignal 状态，保留输入内容方便重试。
当前只做气泡回复，不做完整聊天历史窗口。
```

实现要点：

```text
apps/desktop-pet/src/App.tsx
新增 conversationPanelVisible / conversationInput / conversationPending 状态，右键菜单接入“说话”，输入框发送后调用 Tauri 命令。

apps/desktop-pet/src/styles.css
新增轻量输入框样式，使用高 z-index 避免被拖拽层拦截。

apps/desktop-pet/src-tauri/src/lib.rs
新增 send_desktop_chat_message 命令。命令使用 spawn_blocking 执行本地 TCP HTTP 请求，避免长时间模型回复阻塞前端；并补充响应解析单元测试。
```

## 桌宠对话体验打磨第一阶段

本轮继续打磨右键“说话”后的回复展示体验。这个阶段只做桌宠展示层增强，没有新建数据库表，也没有修改 `kaka-core` 协议。

新增能力：

```text
长回复会按标点和长度拆成连续气泡，而不是直接截断。
每段气泡停留时间会随文本长度轻量延长，短句仍保持轻快。
回复状态会根据文本本地轻量推断：解释/分析类进入思考，积极/确认类进入开心，普通回复进入收到消息。
发送中仍保持 loading 状态。
失败时进入 weakSignal，并按连接失败、超时、响应异常显示不同短句。
失败后保留输入内容，方便用户直接重试。
```

```text

apps/desktop-pet/src/App.tsx
接入连续气泡播放计时器；成功回复时关闭输入框并顺序播放气泡；失败时保留输入框并显示更具体的信号弱提示。
```
