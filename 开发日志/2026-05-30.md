# 2026-05-30 开发日志

## 桌宠生命感增强第一阶段

本轮先做低成本、高收益的生命感增强：继续使用当前 12 张透明 PNG 状态图，不新增素材，不引入骨骼动画，不新建数据库表，也不修改 `kaka-core` 协议。

新增能力：

```text
待机状态加入更轻的呼吸、浮动和小幅摆动。
开心、收到消息、思考、信号弱、炸毛、睡觉和拖拽状态使用不同节奏，减少所有状态动作都一样的机械感。
状态贴图切换时加入轻量弹入和淡入缓动，避免突然硬切。
桌宠大小仍跟随现有窗口大小设置，PixiJS 每帧按贴图高度计算基础缩放。
```

实现要点：

```text
apps/desktop-pet/src/PetCanvas.tsx
新增 MotionFrame，统一描述 x / y / rotation / scaleX / scaleY / alpha。
新增 stateChangedAtRef，记录状态切换时间，用于计算状态切换缓动。
将原本单一 scale 动作拆成横向/纵向缩放，支持轻微 squash/stretch。
新增 getStateTransitionFrame 和 easeOutCubic，给状态切换增加 280ms 的轻量弹入与淡入。
```
