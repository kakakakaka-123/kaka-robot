## 当前素材判断

当前 12 张 1024 透明 PNG 已经能支撑桌宠 v0.1 的状态切换和轻量程序化动作。继续明显提升桌宠质感，重点不应该再补“更多静态状态图”，而应该补“动作序列”。

优先补高频动作：

```text
待机眨眼
睡觉呼吸
摸头开心
收到消息抬头
拖拽放下
```

这些动作比新增大量低频表情更有价值，因为用户长期看到的是待机、摸头、睡觉、收到消息和拖拽。

## 风格识别点

新增素材必须保持这些识别点，不要因为动作帧变化而丢失：

- Q 版大头小身比例。
- 银白色长卷发，带粉紫光泽。
- 粉紫色眼睛，有一点电子光感。
- 白色猫耳，内侧柔和粉色。
- 右侧黑色蝴蝶结。
- 粉色交叉发夹和小心形发饰。
- 黑色颈环和粉色心形吊坠。
- 黑粉白机能风短外套。
- 白色短裤、黑色绑带、粉色腰带。
- 黑粉白高帮鞋 / 厚底鞋。
- 蓬松白色猫尾，尾巴末端有黑色蝴蝶结。
- 可使用聊天气泡、光标、像素感 UI、信号、爱心、Zzz 等小元素，但不能喧宾夺主。

不要跑偏到这些方向：

- 普通粉色猫娘模板。
- 女仆、校园制服或重度赛博朋克。
- 过度性感或成年化。
- 过度复杂机甲。
- 每张都只会甜笑。
- 背景复杂、暗色厚重、不可透明裁切。

## 通用输出规格

默认规格：

```text
格式：PNG
画布：1024 x 1024
背景：透明
色彩：保持现有黑粉白 + 银白粉紫 + 少量蓝紫电子光
角色位置：尽量和当前 1024 透明 PNG 对齐
脚底 / 身体重心：同一动作序列内必须稳定
边缘：保留干净透明通道，不要白底、灰底或绿底残留
```

动作序列要求：

- 同一动作内每一帧画布尺寸一致。
- 同一动作内角色比例一致。
- 角色中心点和脚底位置尽量一致，除非动作明确需要移动。
- 发丝、尾巴、蝴蝶结、鞋子、手部不能突然变形。
- 表情变化要连贯，不要每帧像不同角色。
- 小特效可以动，但不能遮挡脸、发夹、黑蝴蝶结和心形吊坠。

## 第一批动作清单

第一批目标是让卡咔从“状态图会动”变成“桌面上真的活着的小人”。建议先做 5 组。

### 1. idle_blink 待机眨眼

优先级：最高

参考素材：

```text
01_待机_1024透明PNG_v2.png
```

建议帧数：

```text
4-6 帧
```

动作描述：

```text
睁眼 -> 半闭 -> 闭眼 -> 半闭 -> 睁眼
```

注意：

- 只改眼睛和极轻微头发/耳朵，不要改变身体姿势。
- 嘴型可以保持待机原样。
- 这组会长期循环，动作要轻，不要每次都像强表情。

### 2. sleep_loop 睡觉呼吸

优先级：最高

参考素材：

```text
08_睡觉_1024透明PNG_v1.png
```

建议帧数：

```text
6-10 帧
```

动作描述：

```text
抱着黑猫玩偶轻微上下呼吸，头发和尾巴有轻微起伏，Zzz 小符号可轻微浮动。
```

注意：

- 睡觉姿势不要乱改。
- 黑猫玩偶、尾巴蝴蝶结、鞋子位置要稳定。
- 呼吸幅度要小，适合长期循环。

### 3. pet_happy 摸头开心

优先级：最高

参考素材：

```text
09_摸头反应_1024透明PNG_v1.png
22_摸头.png 或 高清1024\22_摸头.png
```

建议帧数：

```text
6-12 帧
```

动作描述：

```text
闭眼开心 -> 小拳头收紧 -> 身体轻轻缩一下 -> 爱心轻微弹出 -> 回到开心停顿。
```

注意：

- 不要画外部手，除非后续明确要做“手摸头”特效。
- 爱心特效可以有，但不要太多。
- 表情应该是被摸头后的开心和有点害羞，不要过度夸张。

### 4. message_pop 收到消息抬头

优先级：高

参考素材：

```text
07_收到消息_1024透明PNG_v1.png
05_收到.png 或 高清1024\05_收到.png
```

建议帧数：

```text
6-10 帧
```

动作描述：

```text
待机 -> 眼睛亮一下 -> 轻轻抬头/举手 -> 聊天气泡或光标弹出 -> 回到收到消息姿态。
```

注意：

- UI 小气泡和光标要保持淡蓝/粉紫电子感。
- 不要让气泡遮脸。
- 这组会用于核心回复和桌宠提醒，表情要清楚但不能太吵。

### 5. drag_drop 拖拽放下

优先级：高

参考素材：

```text
10_拖拽反应_1024透明PNG_v1.png
01_待机_1024透明PNG_v2.png
```

建议帧数：

```text
6-10 帧
```

动作描述：

```text
拖拽姿态 -> 下落/回正 -> 脚底触地 -> 轻微弹一下 -> 回待机。
```

注意：

- 脚底落点要稳定。
- 不要改变整体比例。
- 这组用于松手瞬间，是高感知动作。

## 第二批动作清单

第二批适合在第一批稳定后继续补。

### idle_look_around 待机看旁边

建议帧数：6-10 帧

用途：随机待机小动作。

描述：眼睛向左/右看，耳朵轻动，尾巴轻摆。

### pet_shy 害羞缩一下

建议帧数：6-10 帧

用途：长按、连续摸头、亲近互动。

描述：闭眼、脸红、小幅缩肩，爱心或小星星轻微浮现。

### pet_angry 连续摸头炸毛

建议帧数：8-12 帧

参考：

```text
05_炸毛_1024透明PNG_v1.png
07_炸毛.png 或 高清1024\07_炸毛.png
```

用途：连续摸头过多、长按抗议。

描述：耳朵竖起、头发炸开一点、尾巴翘起、表情不满。

### talk_loop 说话小循环

建议帧数：2-4 帧嘴型，或 6-8 帧轻微身体动作。

用途：对话回复气泡播放期间。

描述：嘴巴轻微开合，身体小幅晃动，避免大幅表情变化。

### loading_loop 加载中循环

建议帧数：6-10 帧

参考：

```text
11_加载中_1024透明PNG_v1.png
04_加载中.png 或 高清1024\04_加载中.png
```

用途：连接测试、发送消息等待回复。

描述：眼神专注，数据/像素点轻微转动或闪烁。

### weak_signal_loop 信号弱循环

建议帧数：6-10 帧

参考：

```text
12_信号弱_1024透明PNG_v1.png
19_信号弱.png 或 高清1024\19_信号弱.png
```

用途：核心未连接、请求失败。

描述：轻微抖动、信号图标闪烁、表情有点无奈。

## 动作接入建议

如果后续素材仍是 PNG 帧序列，建议继续使用 PixiJS，不急着换 Live2D 或 Spine。

建议接入路线：

1. 先把当前 `petStates.ts` 从“单状态单贴图”扩展为“状态 + 可选动画”。

示意：

```ts
{
  id: "idle",
  still: "idle.png",
  animations: {
    blink: ["idle_blink_001.png", "idle_blink_002.png"],
    lookAround: ["idle_look_around_001.png"]
  }
}
```

2. `PetCanvas.tsx` 支持两种播放：

```text
循环动画：sleep_loop / loading_loop / weak_signal_loop
一次性动画：message_pop / pet_happy / drag_drop / wake_up
```

3. 状态调度保持现有优先级：

```text
drag > conversation > system > chatReply > reaction > sleep > ambient > idle
```

4. 动作播放结束后，回到对应状态静帧，不要强制回待机，避免打断当前状态来源。

## 验收标准

每组动作素材交付后，先按下面标准检查：

- 所有帧都是 1024 x 1024。
- 所有帧都有透明 alpha。
- 同组帧角色大小一致。
- 同组帧脚底/身体重心稳定。
- 脸、眼睛、猫耳、发夹、黑蝴蝶结、心形吊坠、尾巴蝴蝶结没有丢。
- 动作连贯，不像每帧换了一个角色。
- 小特效不会遮挡脸。
- 缩到桌宠实际尺寸后仍然看得清表情。
- 可以在 280px 桌宠窗口里识别动作意图。
