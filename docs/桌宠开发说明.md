# 桌宠开发说明

本文档记录卡咔桌宠当前实现、运行方式和下一步计划。

## 当前定位

桌宠是卡咔的桌面身体原型，当前放在：

```text
apps/desktop-pet
```

它暂时独立运行，还没有接入 `kaka-core /v1/chat` 对话。当前目标不是一次性做完整 AI 助手，而是先把桌面身体、状态、基础互动和气泡表达跑通。

## 技术组合

当前桌宠技术组合：

```text
Tauri 2
React 18
Vite
TypeScript
PixiJS 8
```

分工：

- Tauri 负责 Windows 桌面窗口、透明背景、置顶、无边框、原生窗口位置控制和系统托盘。
- React / TypeScript 负责 UI 状态、交互逻辑、右键菜单和气泡。
- Vite 负责前端开发服务器和构建。
- PixiJS 负责透明 PNG 桌宠贴图渲染和轻量动画。

## 当前能力

截至 2026-05-28，桌宠已经完成：

- 透明、无边框、置顶的 `280x280` 桌面窗口。
- 12 个桌宠状态素材：
  - 待机
  - 开心
  - 困困
  - 思考
  - 炸毛
  - 装死404
  - 收到消息
  - 睡觉
  - 摸头反应
  - 拖拽反应
  - 加载中
  - 信号弱
- PixiJS 渲染透明 PNG，并为不同状态提供轻量动作。
- 右键菜单切换状态和退出。
- 右键菜单顶部提供 `连接测试` 和 `重置位置`。
- 系统托盘基础版：
  - 托盘图标。
  - 托盘菜单包含 `显示卡咔` / `隐藏卡咔`、`设置`、`开机自启`、`重置位置`、`连接测试` 和 `退出`。
  - 左键点击托盘图标可显示/隐藏卡咔。
  - 主窗口显示/隐藏后，托盘菜单文字会同步切换。
  - 设置窗口读取或切换开机自启后，托盘里的 `开机自启` 勾选状态会同步刷新。
  - 托盘的 `重置位置` 和 `连接测试` 复用前端已有交互，会正常清理保存坐标和显示气泡。
- 开机自启基础版：
  - 使用官方 `tauri-plugin-autostart`。
  - 托盘菜单里的 `开机自启` 是勾选项。
  - 勾上后注册系统启动项，取消勾选后关闭。
- 设置窗口基础版：
  - 托盘菜单提供 `设置` 入口。
  - 设置窗口是普通小窗口，不透明、不置顶。
  - 可以查看开机自启状态和核心连接状态。
  - 可以开启/关闭开机自启、连接测试、重置位置。
  - 可以设置 `开机自启时显示卡咔`。
  - 可以开关随机待机气泡/小动作。
  - 可以设置闲置睡觉时间：`1 分钟 / 2 分钟 / 5 分钟 / 不自动睡觉`。
- 启动体验优化：
  - 自启动注册会带 `--from-autostart` 参数。
  - 正常手动启动时显示卡咔。
  - 开机自启时默认只驻留托盘，不主动弹出主桌宠窗口。
  - 如果打开 `开机自启时显示卡咔`，自启后也会显示主桌宠窗口。
- 连接测试会检查 `kaka-core /health`，只验证本地核心服务是否在线，不调用大模型。
- 重置位置会清掉保存的窗口坐标，并把桌宠移回屏幕中间。
- 窗口位置记忆：拖动后保存窗口坐标，下次启动自动恢复。
- 左键轻点触发摸头反应，约 2 秒后回到待机。
- 左键拖动移动桌宠窗口，拖拽时显示拖拽反应，松手后恢复拖拽前状态。
- 2 分钟无操作后自动进入睡觉状态。
- 睡觉状态下点击、拖动或右键会唤醒。
- 随机待机气泡：待机一段随机时间后偶尔冒出短句。
- 随机待机小动作：待机时偶尔短暂切到开心、思考或困困，然后回到待机。
- 消息气泡雏形：
  - 摸头时显示短句。
  - 闲置睡觉时显示短句。
  - 睡醒时显示短句。
  - 手动切状态时显示对应短句。

## 运行方式

第一次运行前需要安装 Node 依赖：

```bat
cd /d D:\Python\AgentRobot\kaka-v2\apps\desktop-pet
npm install
```

开发运行：

```bat
cd /d D:\Python\AgentRobot\kaka-v2\apps\desktop-pet
npm run tauri:dev
```

只检查前端构建：

```bat
cd /d D:\Python\AgentRobot\kaka-v2\apps\desktop-pet
npm run build
```

检查 Rust / Tauri 侧：

```bat
cd /d D:\Python\AgentRobot\kaka-v2\apps\desktop-pet\src-tauri
cargo check
```

如果 PyCharm 或终端提示找不到 `cargo`，先确认 Rust 已安装，并把下面目录加入当前终端 PATH：

```bat
C:\Users\fushunshun\.cargo\bin
```

PowerShell 临时补 PATH：

```powershell
$env:Path="$env:USERPROFILE\.cargo\bin;$env:Path"
```

## 注意事项

- `node_modules/`、`dist/` 和 `src-tauri/target/` 都是生成产物，不应该提交。
- 当前桌宠窗口是 `280x280`，右键菜单会遮挡卡咔本体，这是现阶段可接受的调试菜单限制。
- 不建议为了菜单不遮挡直接把透明窗口做很大，因为透明区域也可能拦截桌面点击。
- 当前已有系统托盘基础入口；后续如果要放更多选项，优先做单独设置窗口，不继续把右键菜单撑大。
- 当前气泡主要是本地 UI 雏形，还没有调用 `kaka-core /v1/chat`。
- `连接测试` 只检查 `kaka-core /health`，用于确认桌宠能找到核心服务；它不代表已经连接或调用大模型。

## 下一步

下一步有两条路线：

```text
体验优先：系统托盘、显示/隐藏、开机自启、设置窗口。
大脑接入：桌宠气泡/输入接入 kaka-core /v1/chat。
```

其中系统托盘基础版、托盘菜单状态优化、开机自启基础版和设置窗口基础版已经完成，体验优先路线下一步建议做：

```text
设置窗口细节打磨
开机自启真实重启回归测试
```

如果后续要接入 `kaka-core` 对话，建议流程是：

```text
桌宠气泡/输入
-> 组装 platform=desktop 的 MessageEvent
-> POST http://127.0.0.1:8001/v1/chat
-> 显示 KakaResponse 的文本到气泡
-> 根据响应或事件切换桌宠状态
```

建议使用的桌宠消息事件：

```json
{
  "platform": "desktop",
  "scene_type": "private",
  "scene_id": "desktop-local",
  "user_id": "owner",
  "display_name": "主人",
  "content": {
    "type": "text",
    "text": "卡咔，在吗"
  }
}
```

接入前先保持简单：先做一个固定输入或小输入框，不急着做完整聊天窗口。当前阶段如果暂时不需要桌宠对话能力，优先继续完善系统托盘、设置、显示/隐藏和开机自启。
