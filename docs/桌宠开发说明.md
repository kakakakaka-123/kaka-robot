# 桌宠开发说明

本文档记录卡咔桌宠当前实现、运行方式和下一步计划。

## 当前定位

桌宠是卡咔的桌面身体原型，当前放在：

```text
apps/desktop-pet
```

它当前已经具备桌面身体、状态、基础互动、气泡表达和最小对话入口。桌宠可以通过右键菜单或双击打开轻量输入框，并经 Tauri 后端调用本机 `kaka-core /v1/chat`，把核心回复分段显示到气泡里。

## 技术组合

当前桌宠技术组合：

```text
Tauri 2
React 18
Vite
TypeScript
PixiJS 8
```

分工：

- Tauri 负责 Windows 桌面窗口、透明背景、置顶、无边框、原生窗口位置控制和系统托盘。
- React / TypeScript 负责 UI 状态、交互逻辑、右键菜单和气泡。
- Vite 负责前端开发服务器和构建。
- PixiJS 负责透明 PNG 桌宠贴图渲染和轻量动画。

## 当前能力

截至 2026-05-30，桌宠已经完成：

- 透明、无边框、置顶的 `280x280` 桌面窗口。
- 12 个桌宠状态素材：
  - 待机
  - 开心
  - 困困
  - 思考
  - 炸毛
  - 装死404
  - 收到消息
  - 睡觉
  - 摸头反应
  - 拖拽反应
  - 加载中
  - 信号弱
- PixiJS 渲染透明 PNG，并为不同状态提供轻量动作：
  - 待机有轻微呼吸、浮动和小幅摆动。
  - 开心、收到消息、思考、信号弱、炸毛、睡觉和拖拽各自有不同节奏。
  - 状态贴图切换时有轻量弹入和淡入缓动，避免突然换图。
- 状态调度已有来源优先级：
  - 拖拽、对话、系统检查、回复播放、普通互动、睡觉和随机待机不会随意互相覆盖。
  - 旧 timer 恢复状态时会校验来源，避免把当前正在进行的对话、拖拽或系统状态切回待机。
  - 对话回复会先短暂进入 `收到消息`，再按内容切到 `思考`、`开心` 或继续 `收到消息`，最后轻停一下再回待机。
- 右键菜单收敛：
  - 默认保留 `说话`、`摸摸头`、`设置`、`隐藏` 和 `退出`。
  - 调试状态菜单默认关闭。
  - 在设置窗口打开 `启用右键调试状态菜单` 后，右键菜单才显示 `连接测试`、`重置位置` 和 12 个状态切换按钮。
- 轻量对话入口：
  - 右键菜单点击 `说话` 会打开小输入框。
  - 双击卡咔会直接打开小输入框；输入框已打开时，双击会重新聚焦输入框。
  - 输入框限制 80 字，支持发送、取消和 Esc 关闭。
  - 输入框会自动聚焦，Enter 发送，Esc 关闭；中文输入法组合候选时不会误触发发送。
  - 发送时桌宠切到加载状态，按钮进入发送中状态；失败后按钮会显示 `重试`，输入内容会保留。
  - 打开输入框、空输入、发送中和失败提示都显示在输入面板内部，避免和卡咔头上的气泡抢位置。
  - 小输入框上方会显示本次运行内最近 6 条本地对话记录，只做临时回看，不写入数据库；最近对话默认折叠，需要时可展开查看完整内容。
  - 输入框打开或发送中时，不会被随机待机和闲置睡觉打断。
- 最小 `kaka-core /v1/chat` 接入：
  - 前端通过 Tauri 命令 `send_desktop_chat_message` 发送文本。
  - Rust 后端组装 `platform=desktop` 的 `MessageEvent`，请求 `http://127.0.0.1:8001/v1/chat`。
  - 返回后从 `KakaResponse.actions` 里取第一条 `send_text` 文本显示到桌宠气泡。
  - 回复会按标点和长度拆成连续气泡，气泡停留时间会随文本长度轻量延长。
  - 回复状态在桌宠本地按文本轻量推断：长回复或解释/分析类偏 `思考`，明显开心/撒娇类偏 `开心`，短普通确认偏 `收到消息`，避免状态跳得太频繁。
  - 回复状态有轻量衔接：成功返回后先短暂显示收到消息，再进入内容状态；播放结束后短暂停留，再自然回到待机。
  - 如果核心未启动或请求失败，桌宠切到 `信号弱` 状态并保留输入内容，方便重试。
  - 失败提示会区分连接失败、超时和响应异常，避免所有错误都显示成同一句。
  - 当前只做最小气泡回复，没有完整聊天历史窗口。
- 系统托盘基础版：
  - 托盘图标。
  - 托盘菜单包含 `显示卡咔` / `隐藏卡咔`、`设置`、`开机自启`、`重置位置`、`修复窗口`、`连接测试`、`重启桌宠` 和 `退出`。
  - 左键点击托盘图标可显示/隐藏卡咔。
  - 主窗口显示/隐藏后，托盘菜单文字会同步切换。
  - 设置窗口读取或切换开机自启后，托盘里的 `开机自启` 勾选状态会同步刷新。
  - 托盘的 `重置位置` 和 `连接测试` 复用前端已有交互，会正常清理保存坐标和显示气泡。
  - 托盘的 `修复窗口` 会隐藏设置窗口、显示并居中桌宠主窗口。
  - 托盘的 `重启桌宠` 会启动当前 exe 的新进程，并退出当前进程。
- 开机自启基础版：
  - 使用官方 `tauri-plugin-autostart`。
  - 托盘菜单里的 `开机自启` 是勾选项。
  - 勾上后注册系统启动项，取消勾选后关闭。
- 设置窗口基础版：
  - 托盘菜单提供 `设置` 入口。
  - 设置窗口是普通小窗口，不透明、不置顶。
  - 可以查看桌宠主窗口当前显示/隐藏状态。
  - 可以查看开机自启状态和核心连接状态。
  - 可以显示/隐藏卡咔主窗口。
  - 可以退出卡咔。
  - 可以开启/关闭开机自启、连接测试、重置位置。
  - 可以设置 `开机自启时显示卡咔`。
  - 可以开关随机待机气泡/小动作。
  - 可以开关右键调试状态菜单。
  - 可以设置桌宠大小：`小 / 标准 / 大`，会同步调整主窗口尺寸和 Pixi 本体渲染比例。
  - 可以设置气泡停留时间：`短 / 标准 / 长`。
  - 可以设置随机待机频率：`低 / 标准 / 高`。
  - 可以开关 `桌宠始终置顶`。
  - 可以开关顶部状态标签。
  - 可以设置闲置睡觉时间：`1 分钟 / 2 分钟 / 5 分钟 / 不自动睡觉`。
  - 设置窗口里的 `连接测试` 会同步通知桌宠主窗口，卡咔本体会显示检查中、核心信号正常或信号弱，并按状态调度规则自动恢复。
- 设置窗口细节打磨：
  - 状态值使用更清楚的颜色标签。
  - 开机自启、连接测试、重置位置和启动显示设置执行时会显示处理中状态。
  - 操作执行中会临时禁用按钮和控件，避免重复点击。
  - 成功、失败和检查中的提示使用不同样式。
- 设置窗口加载兜底：
  - 设置窗口使用独立 `settings.html` 入口，不再和桌宠主窗口共用 `index.html`。
  - 设置窗口由 Tauri 启动时预创建并默认隐藏，打开设置只显示已有窗口。
  - 设置窗口右上角关闭按钮会隐藏窗口，不销毁 WebView。
  - 前端入口加入错误兜底页，运行时异常不再只显示空白窗口。
- 启动体验优化：
  - 自启动注册会带 `--from-autostart` 参数。
  - 正常手动启动时显示卡咔。
  - 开机自启时默认只驻留托盘，不主动弹出主桌宠窗口。
  - 如果打开 `开机自启时显示卡咔`，自启后也会显示主桌宠窗口。
- 连接测试会检查 `kaka-core /health`，只验证本地核心服务是否在线，不调用大模型。
- 重置位置会清掉保存的窗口坐标，并把桌宠移回屏幕中间。
- 窗口位置记忆：拖动后保存窗口坐标，下次启动自动恢复。
- 已新增本地行为系统：
  - `petBehavior.ts` 统一管理摸头、双击、长按、拖拽、待机、睡醒和睡觉气泡。
  - 行为反应支持权重、防止连续重复同一句、单条反应冷却和连续摸头计数。
- 左键轻点触发摸头反应，会根据权重、冷却和连续摸头次数出现摸头、开心、思考或炸毛状态，并显示不同短句。
- 左键双击会触发更强一点的确认/在线反馈。
- 左键长按会触发观察、害羞或轻微抗议类反应。
- 左键拖动移动桌宠窗口，拖拽时显示拖拽反应，松手后恢复拖拽前状态。
- 拖拽时间较长时会显示特殊气泡，避免长时间拖拽只有静态反馈。
- 拖拽松手后会显示轻量反馈气泡。
- 2 分钟无操作后自动进入睡觉状态。
- 睡觉状态下点击、拖动或右键会唤醒，并随机显示短句。
- 随机待机气泡：待机一段随机时间后偶尔冒出短句。
- 随机待机小动作：待机时偶尔短暂切到开心、思考或困困，然后回到待机。
- 消息气泡有轻量弹入动画。
- 消息气泡雏形：
  - 摸头时显示短句。
  - 闲置睡觉时显示短句。
  - 睡醒时显示短句。
  - 手动切状态时显示对应短句。
  - `kaka-core /v1/chat` 成功返回时显示核心回复，长回复会拆成连续气泡，超过上限时追加兜底提示。
  - 对话回复会按内容轻量推断 `思考`、`开心` 或 `收到消息` 状态，短普通回复不会频繁切开心。
  - 对话输入框内保留最近几条本地临时对话，默认折叠，展开后可完整回看。

## 运行方式

第一次运行前需要安装 Node 依赖：

```bat
cd /d D:\Python\AgentRobot\kaka-v2\apps\desktop-pet
npm install
```

开发运行：

```bat
cd /d D:\Python\AgentRobot\kaka-v2\apps\desktop-pet
npm run tauri:dev
```

只检查前端构建：

```bat
cd /d D:\Python\AgentRobot\kaka-v2\apps\desktop-pet
npm run build
```

```bat
cd /d D:\Python\AgentRobot\kaka-v2\apps\desktop-pet\src-tauri
cargo check
```

如果 PyCharm 或终端提示找不到 `cargo`，先确认 Rust 已安装，并把下面目录加入当前终端 PATH：

```bat
C:\Users\fushunshun\.cargo\bin
```

PowerShell 临时补 PATH：

```powershell

```

## 注意事项

- `node_modules/`、`dist/` 和 `src-tauri/target/` 都是生成产物，不应该提交。
- 当前桌宠窗口是 `280x280`，默认右键菜单已经收敛成轻量操作；调试状态菜单打开后仍可能遮挡卡咔本体，这是开发测试入口的可接受限制。
- 不建议为了菜单不遮挡直接把透明窗口做很大，因为透明区域也可能拦截桌面点击。
- 当前已有系统托盘基础入口；后续如果要放更多选项，优先做单独设置窗口，不继续把右键菜单撑大。
- 当前对话入口只做最小气泡回复，没有完整聊天记录、上下文展示、快捷键唤醒或独立聊天面板。
- `连接测试` 只检查 `kaka-core /health`，用于确认桌宠能找到核心服务；真正发送输入框内容时才会调用 `/v1/chat`，如果核心启用了 LLM 且配置了 Key，这一步才可能触发大模型。
