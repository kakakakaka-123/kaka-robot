# 卡咔插件开发指南

## 概述

卡咔的插件系统允许你扩展卡咔的能力，而无需修改核心代码。

**插件可以做什么**：
- 匹配特定消息（关键词、正则、LLM 判断）
- 执行自定义逻辑（调用外部 API、数据库操作）
- 返回回复文本或执行无回复操作

---

## 插件协议

所有插件必须实现 `KakaPlugin` 协议：

```python
from kaka_core.plugins.context import PluginContext
from kaka_core.plugins.result import PluginResult


class MyPlugin:
    """我的自定义插件。"""

    id = "my_plugin"  # 唯一标识
    name = "我的插件"  # 显示名称
    description = "插件功能描述"  # 描述

    async def can_handle(self, context: PluginContext) -> bool:
        """判断是否可以处理该消息。"""
        return "关键词" in context.text

    async def run(self, context: PluginContext) -> PluginResult:
        """执行插件逻辑。"""
        # 处理消息
        result = do_something(context.text)

        # 返回回复
        return PluginResult.text_reply(
            self.id,
            f"处理结果：{result}"
        )
```

---

## PluginContext 上下文

插件通过 `PluginContext` 获取消息信息：

```python
@dataclass
class PluginContext:
    event_id: str          # 事件 ID
    platform: str          # 平台（qq/desktop/web）
    scene_type: str        # 场景类型（private/group）
    scene_id: str          # 场景 ID
    user_id: str           # 用户 ID
    display_name: str      # 用户昵称
    text: str              # 消息文本
    content_type: str      # 内容类型
    command_text: str      # 命令文本（如果是命令模式）
    metadata: dict         # 额外元数据
```

**常用字段**：
- `context.text` - 消息文本内容
- `context.user_id` - 发送者用户 ID
- `context.platform` - 来源平台
- `context.scene_id` - 场景 ID（群号/私聊 ID）

---

## PluginResult 返回值

插件执行后返回 `PluginResult`：

### 1. 文本回复

```python
return PluginResult.text_reply(
    plugin_id=self.id,
    text="回复内容"
)
```

### 2. 无回复

```python
return PluginResult.no_reply(self.id)
```

### 3. 带元数据的回复

```python
return PluginResult.text_reply(
    self.id,
    "处理完成",
    metadata={"task_id": 123}
)
```

## 实战示例

### 示例 1：天气查询插件

```python
"""天气查询插件。"""

import httpx

from kaka_core.plugins.context import PluginContext
from kaka_core.plugins.result import PluginResult


class WeatherPlugin:
    """天气查询插件。"""

    id = "weather"
    name = "天气查询"
    description = "查询城市天气"

    async def can_handle(self, context: PluginContext) -> bool:
        """匹配天气查询消息。"""
        keywords = ["天气", "温度", "下雨吗", "要带伞吗"]
        return any(kw in context.text for kw in keywords)

    async def run(self, context: PluginContext) -> PluginResult:
        """执行天气查询。"""
        # 提取城市名
        city = self._extract_city(context.text)
        if not city:
            return PluginResult.text_reply(
                self.id,
                "你想查哪个城市的天气呀？"
            )

        # 调用天气 API
        weather = await self._fetch_weather(city)
        if not weather:
            return PluginResult.text_reply(
                self.id,
                f"{city}的天气查不到呢..."
            )

        # 构造回复
        reply = f"{city}今天{weather['condition']}，{weather['temp']}°C"
        return PluginResult.text_reply(self.id, reply)

    def _extract_city(self, text: str) -> str:
        """从文本提取城市名。"""
        # 简单实现：假设"北京天气"格式
        for city in ["北京", "上海", "广州", "深圳"]:
            if city in text:
                return city
        return ""

    async def _fetch_weather(self, city: str) -> dict | None:
        """调用天气 API。"""
        try:
            async with httpx.AsyncClient() as client:
                # 示例 API（需替换为真实 API）
                response = await client.get(
                    f"https://api.weather.com/v1/{city}"
                )
                return response.json()
        except Exception:
            return None
```

### 示例 2：计算器插件

```python
"""简单计算器插件。"""

import re

from kaka_core.plugins.context import PluginContext
from kaka_core.plugins.result import PluginResult


class CalculatorPlugin:
    """计算器插件。"""

    id = "calculator"
    name = "计算器"
    description = "简单数学计算"

    async def can_handle(self, context: PluginContext) -> bool:
        """匹配计算表达式。"""
        # 匹配 "1+1" "3*5" 等
        pattern = r'\d+\s*[\+\-\*/]\s*\d+'
        return bool(re.search(pattern, context.text))

    async def run(self, context: PluginContext) -> PluginResult:
        """执行计算。"""
        # 提取表达式
        pattern = r'(\d+)\s*([\+\-\*/])\s*(\d+)'
        match = re.search(pattern, context.text)
        if not match:
            return PluginResult.no_reply(self.id)

        a, op, b = match.groups()
        a, b = int(a), int(b)

        # 计算
        if op == '+':
            result = a + b
        elif op == '-':
            result = a - b
        elif op == '*':
            result = a * b
        elif op == '/':
            if b == 0:
                return PluginResult.text_reply(self.id, "不能除以0哦")
            result = a / b
        else:
            return PluginResult.no_reply(self.id)

        return PluginResult.text_reply(
            self.id,
            f"{a} {op} {b} = {result}"
        )
```

### 示例 3：记忆查询插件

```python
"""记忆查询插件（内置）。"""

from kaka_core.memory.search import search_user_memories, MemorySearchFilters
from kaka_core.plugins.context import PluginContext
from kaka_core.plugins.result import PluginResult
from kaka_core.storage.database import create_session_factory, init_database


class MemorySearchPlugin:
    """记忆查询插件。"""

    id = "memory_search"
    name = "记忆查询"
    description = "查询长期记忆"

    def __init__(self, session_factory=None):
        self._session_factory = session_factory

    async def can_handle(self, context: PluginContext) -> bool:
        """通过命令触发：/记忆 或 插件：记忆。"""
        return bool(context.command_text)

    async def run(self, context: PluginContext) -> PluginResult:
        """查询记忆。"""
        query = context.command_text.strip()
        if not query:
            return PluginResult.text_reply(
                self.id,
                "要查什么记忆，得先给卡咔一个关键词。"
            )

        # 初始化数据库
        session_factory = self._session_factory
        if not session_factory:
            init_database()
            session_factory = create_session_factory()

        # 查询记忆
        with session_factory() as session:
            results = search_user_memories(
                session,
                MemorySearchFilters(
                    platform=context.platform,
                    user_id=context.user_id,
                    query_text=query,
                    limit=5,
                ),
            )

        # 构造回复
        if not results:
            return PluginResult.text_reply(
                self.id,
                f"没找到关于「{query}」的记忆..."
            )

        reply = f"找到 {len(results)} 条记忆：\n"
        for i, memory in enumerate(results, 1):
            reply += f"{i}. {memory.content}\n"

        return PluginResult.text_reply(self.id, reply.strip())
```

## 最佳实践

### 1. 明确匹配条件

避免过于宽泛的匹配：

```python
# ❌ 不好：匹配所有消息
async def can_handle(self, context):
    return True

# ✅ 好：明确匹配条件
async def can_handle(self, context):
    return "天气" in context.text
```

### 2. 异常处理

插件内部应捕获异常：

```python
async def run(self, context):
    try:
        result = await external_api()
        return PluginResult.text_reply(self.id, result)
    except Exception as e:
        return PluginResult.text_reply(
            self.id,
            f"出错了：{str(e)}"
        )
```

### 3. 避免阻塞

使用异步 I/O：

```python
# ✅ 好：异步 HTTP
async with httpx.AsyncClient() as client:
    response = await client.get(url)

# ❌ 不好：同步阻塞
import requests
response = requests.get(url)  # 阻塞事件循环
```

### 4. 数据库会话管理

使用 context manager：

```python
from kaka_core.storage.database import create_session_factory

session_factory = create_session_factory()

with session_factory() as session:
    # 数据库操作
    result = session.query(...)
    session.commit()
# 自动关闭
```

### 5. 测试插件

编写单元测试：

```python
import pytest
from kaka_protocol import Platform, SceneType
from kaka_core.plugins.context import PluginContext
from my_plugin import MyPlugin


@pytest.mark.asyncio
async def test_my_plugin():
    """测试插件功能。"""
    plugin = MyPlugin()

    # 构造上下文
    context = PluginContext(
        event_id="test",
        platform="qq",
        scene_type="private",
        scene_id="user_123",
        user_id="user_123",
        display_name="测试",
        text="触发关键词",
        content_type="text",
        command_text="",
        metadata={},
    )

    # 测试匹配
    can_handle = await plugin.can_handle(context)
    assert can_handle

    # 测试执行
    result = await plugin.run(context)
    assert result.reply_text is not None
```

## 常见问题

### Q: 插件优先级如何控制？

A: 插件按注册顺序执行，第一个匹配的插件会处理消息。调整 `PluginRegistry` 中的顺序即可。

### Q: 一个消息可以被多个插件处理吗？

A: 当前实现是第一个匹配的插件处理。如需多插件，可以在插件内部调用其他插件。

### Q: 如何访问外部配置？

A: 使用 `get_settings()`：

```python
from kaka_core.config.settings import get_settings

settings = get_settings()
api_key = settings.some_api_key
```

### Q: 插件可以访问数据库吗？

A: 可以，通过 `create_session_factory()`：

```python
from kaka_core.storage.database import create_session_factory

session_factory = create_session_factory()
with session_factory() as session:
    # 数据库操作
    pass
```
