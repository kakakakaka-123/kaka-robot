# 卡咔 v2 统一消息协议

本文档说明 `packages/kaka-protocol` 的第一版设计。

这一层的目的不是连接 QQ，也不是调用大模型，而是定义一套 `qq-adapter`、未来网页聊天端、`iot-adapter`、`voice-gateway` 和 `kaka-core` 都能理解的数据格式。

注意：当前已实现的 `apps/web-console` 是本地管理台，主要调用 `/admin/api/*` 管理数据；它不是面向普通聊天的 Web 入口。未来如果新增网页聊天端，也应该继续使用这里定义的统一消息协议。

## 为什么需要协议层

不同平台的原始消息格式不同。

QQ 群聊消息可能包含：

```text
群号
QQ 号
群昵称
消息文本
消息 ID
```

网页聊天消息可能包含：

```text
房间 ID
用户 ID
用户名
输入内容
```

IoT 设备事件可能包含：

```text
设备 ID
传感器类型
传感器数值
事件时间
```

`kaka-core` 不应该为每个平台写一套处理逻辑。所有入口都应该先把原始消息转换成统一协议，再交给核心大脑。

## 基本流程

```text
QQ / Web / Voice / IoT 原始输入
-> Adapter 适配器
-> MessageEvent
-> Kaka Core
-> KakaResponse
-> Adapter 适配器
-> 发回对应平台
```

## MessageEvent

`MessageEvent` 是外部输入进入 `kaka-core` 的统一格式。

核心字段：

```text
event_id      事件唯一 ID
platform      来源平台，例如 qq / web / voice / iot
scene_type    场景类型，例如 private / group / room / device
scene_id      场景 ID，例如群号、房间号、设备 ID
user_id       用户唯一 ID
display_name  当前显示名
content       消息内容
timestamp     事件时间
raw_event     原始平台事件
metadata      额外信息
```

示例：

```json
{
  "platform": "qq",
  "scene_type": "group",
  "scene_id": "20002",
  "user_id": "10001",
  "display_name": "群友A",
  "content": {
    "type": "text",
    "text": "卡咔在吗"
  }
}
```

## MessageContent

`MessageContent` 表示统一后的内容。

核心字段：

```text
type       内容类型，例如 text / image / voice / video / sensor
text       文本内容
media_url  媒体文件地址
mime_type  媒体类型
data       结构化数据
```

第一阶段主要使用 `text`。

后续图片、语音、视频、传感器数据都可以继续扩展到这个结构里。

## KakaResponse

`KakaResponse` 是 `kaka-core` 返回给适配器的统一格式。

核心字段：

```text
response_id   回复唯一 ID
event_id      对应的输入事件 ID
should_reply  是否需要回复
actions       要执行的动作列表
created_at    创建时间
metadata      额外信息
```

示例：

```json
{
  "event_id": "event-1",
  "should_reply": true,
  "actions": [
    {
      "type": "send_text",
      "content": {
        "type": "text",
        "text": "我在。"
      }
    }
  ]
}
```

## ResponseAction

`ResponseAction` 表示适配器需要执行的一个动作。

第一版支持：

```text
send_text   发送文本
send_image  发送图片
send_voice  发送语音
send_video  发送视频
send_emoji  发送表情
call_tool   调用工具
noop        不执行动作
```

第一阶段实际主要使用 `send_text`。

## 当前边界

当前协议层只负责定义格式。

它不负责：

- 连接 QQ。
- 登录 QQ。
- 调用 LLM。
- 保存数据库。
- 判断是否回复。
- 生成人格化文本。

这些分别由 `qq-adapter` 和 `kaka-core` 实现。

## 当前接入状态

协议层已经被当前真实链路使用：

```text
QQ 文本消息
-> qq-adapter 转 MessageEvent
-> kaka-core 生成 KakaResponse
-> qq-adapter 转 QQSendTextAction
-> QQ 发出文本回复
```

当前第一阶段主要使用：

- `MessageEvent.content.type = text`
- `ResponseAction.type = send_text`

下一步如果要扩展图片、语音、表情包，不应该绕开协议层，而应该继续扩展 `MessageContent` 和 `ResponseAction` 的使用方式。
