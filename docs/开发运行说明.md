## 数据库记录

当前 `kaka-core` 会把基础消息和回复记录写入 SQLite。

默认数据库文件：

```text
D:\Python\AgentRobot\kaka-v2\data\kaka.sqlite3
```

当前表：

```text
users
scenes
inputs
outputs
memory_candidates
memories
```

当前表语义：

```text
users：外部平台用户。
scenes：私聊、群聊等消息场景。
inputs：卡咔收到/观察到的输入，包含 analysis_status。
outputs：卡咔对已处理输入形成的输出结果或响应决策，包含 output_origin、output_reason、no_reply_reason；普通观察消息不会写入 outputs。
memory_candidates：长期记忆候选区，候选先在这里等待审核、拒绝、标记重复或合并。
memories：正式长期记忆表，回复前只检索 active 记忆。
```

当前状态语义：

```text
inputs.analysis_status：
  not_analyzed：还没有被长期记忆分析处理
  analyzed：已经处理过，或已经产生候选/被人工标记为已处理
  skipped：确认没有长期记忆价值，后续不再分析

memory_candidates.status：
  pending：等待审核或合并
  approved：已经合并进正式 memories
  rejected：确认不进入正式记忆
  merged_duplicate：和已有正式记忆重复，没有新增 memories

memories.status：
  active：参与回复前检索
  archived：保留但不参与回复
```

时间说明：

```text
created_at 当前按 UTC 存储。
在 Navicat 里直接看会比北京时间少 8 小时。
例如数据库里 2026-04-30 17:40:49，对应北京时间 2026-05-01 01:40:49。
```

数据库文件包含真实聊天记录，已经被 `.gitignore` 忽略，不应该提交。

如果需要换数据库地址，可以设置：

```env
DATABASE_URL=sqlite:///D:/Python/AgentRobot/kaka-v2/data/kaka.sqlite3
```

## 分析未处理输入

可以使用最小分析脚本查看还没有进入后续分析流程的输入：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py
```

脚本默认只读预览，不修改数据库。

如果在 PyCharm 中右键运行，不需要配置 Parameters。直接修改脚本顶部的：

```python
PYCHARM_INPUT_IDS = ""          # 已知 inputs.id 时填写，例如 "12,13"
PYCHARM_LIMIT = 30
PYCHARM_GROUP_ID = ""
PYCHARM_USER_ID = ""
PYCHARM_LLM_BATCH = False
PYCHARM_WRITE_CANDIDATES = False
PYCHARM_MARK_SKIPPED = False
```

默认只读预览。`PYCHARM_WRITE_CANDIDATES=True` 必须配合 `PYCHARM_LLM_BATCH=True`；确认输出没问题后再打开写库开关。

常用筛选：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --limit 50
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --group 1073224364
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --user 1419825488
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --date 2026-05-01
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --private
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --group-chat
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --llm-batch
```

当前分类：

```text
skipped：明显无价值或占位消息，例如寒暄、测试消息、@ 占位
candidate：可能值得后续进入记忆候选，例如身份、偏好、项目长期信息
not_sure：普通聊天，暂时不自动跳过
```

批量调用记忆模型做只读判断：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --limit 50 --llm-batch
```

确认输出没问题后，写入候选区：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --limit 50 --llm-batch --write-candidates
```

`--write-candidates` 会修改数据库：

```text
LLM/rule candidate -> 写入 memory_candidates，来源 input 标记 analyzed
LLM/rule skipped   -> 来源 input 标记 skipped
LLM error / 缺结果 -> 保持 not_analyzed
```

如果确认明显无价值记录可以跳过，可以加：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\analyze_inputs.py --mark-skipped
```

`--mark-skipped` 只会把脚本判定为 `skipped` 的记录更新为：

```text
analysis_status=skipped
```

不会写入长期记忆，也不会修改 `outputs`。

查看候选区：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\show_memory_candidates.py
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\show_memory_candidates.py --status all --limit 50
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\show_memory_candidates.py --type user_fact --limit 30
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\show_memory_candidates.py --ids 12,13
```

PyCharm 里可以直接填：

```python
PYCHARM_CANDIDATE_IDS = "候选ID"
PYCHARM_STATUS = ""  # 填了 ID 时默认查 all；不填 ID 时默认查 pending
```

把候选区合并到正式长期记忆：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\merge_memory_candidates.py --limit 20
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\merge_memory_candidates.py --limit 20 --apply
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\merge_memory_candidates.py --ids 12,13 --apply
```

不加 `--apply` 时只读预览；加 `--apply` 后写入 `memories`，成功合并的候选标记为 `approved`，重复候选标记为 `merged_duplicate`。

如果已经在数据库可视化软件里确认了候选 ID，更推荐打开脚本填：

```python
PYCHARM_CANDIDATE_IDS = "候选ID"
PYCHARM_APPLY = False  # 先预览；确认后改 True
```

测试阶段如果没有时间人工检查候选，可以让 LLM 临时代替人工复核：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\review_memory_candidates.py --limit 20
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\review_memory_candidates.py --limit 20 --apply
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\review_memory_candidates.py --ids 12,13 --apply
```

不加 `--apply` 时只读预览；加 `--apply` 后按 approve/reject/duplicate 更新候选，并把通过的候选写入正式 `memories`。

PyCharm 里同样可以只填：

```python

PYCHARM_APPLY = False  # 先预览；确认后改 True
```

管理正式长期记忆：

如果已经在数据库可视化软件里看到 `memories.id`，更推荐直接打开：

```text
services/kaka-core/scripts/manage_memories.py
```

然后在文件顶部改简单配置：

```python
PYCHARM_MEMORY_IDS = "记忆ID"
PYCHARM_ACTION = "archive"   # archive / restore / delete
PYCHARM_APPLY = False        # 先预览；确认后改 True
```

常用命令行方式仍然可用：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\manage_memories.py --id 记忆ID --status archived
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\manage_memories.py --id 记忆ID --status archived --apply
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\manage_memories.py --id 记忆ID --status active --apply
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\manage_memories.py --id 记忆ID --delete --apply --yes
```

`active` 参与回复检索，`archived` 保留但不参与回复。硬删除只用于确认错误、垃圾或敏感记忆。

预览正式长期记忆检索：

```powershell
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\search_memories.py --user QQ号 --text "当前消息文本"
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\search_memories.py --user QQ号 --group 群号 --text "当前消息文本"
.\.venv\Scripts\python.exe .\services\kaka-core\scripts\search_memories.py --user QQ号 --private --text "当前消息文本"
```

`search_memories.py` 只读读取 `memories`，按当前用户、当前消息、可选场景打分，打印少量命中记忆和命中原因，不会修改数据库，也不会影响当前聊天回复。

PyCharm 里直接填：

```python
PYCHARM_USER_ID = "QQ号"
PYCHARM_TEXT = "当前消息文本"
PYCHARM_GROUP_ID = ""   # 群聊填群号；私聊则把 PYCHARM_PRIVATE 改 True
PYCHARM_PRIVATE = False
```

如果在 `.env` 开启自动候选分析：

```env
MEMORY_AUTO_ANALYSIS_ENABLED=true
MEMORY_AUTO_ANALYSIS_TRIGGER_COUNT=50
MEMORY_AUTO_ANALYSIS_BATCH_LIMIT=50
MEMORY_AUTO_ANALYSIS_MAX_RUNS_PER_CHECK=2
```

`kaka-core` 启动后不会立刻分析，会读取本机时间，等到下一个整点检查；之后每个整点检查一次。只有 `not_analyzed >= 50` 时才调用大模型，每个整点最多跑两轮。

如果在 `.env` 开启自动候选区复核：

```env
MEMORY_AUTO_REVIEW_ENABLED=true
MEMORY_AUTO_REVIEW_TRIGGER_COUNT=20
MEMORY_AUTO_REVIEW_BATCH_SIZE=10
MEMORY_AUTO_REVIEW_MAX_RUNS_PER_CHECK=1
```

`kaka-core` 启动后同样等到下一个整点检查。只有 `memory_candidates.pending >= 20` 时才调用大模型；每批 10 条，每个整点最多跑一批。

当前页面能力：

