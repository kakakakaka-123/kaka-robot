# 卡咔桌面操作能力说明

## 功能概述

卡咔现在拥有在主人电脑上执行操作的能力：
- ✅ 在桌面创建文件和小纸条
- ⏳ 截图分享（待实现）
- ⏳ 播放音效（待实现）

**重要设计原则**：这些能力是卡咔本身的能力，对外不暴露"助手"概念。

---

## 架构设计

```
QQ 群友发指令 "在桌面写个文件"
  ↓
kaka-core 插件系统解析
  ↓
DesktopOperationsPlugin 创建任务
  ↓
数据库 desktop_operations 表 (status=pending)
  ↓
kaka-local 本地组件轮询获取任务
  ↓
执行操作（创建文件）
  ↓
上传结果到 kaka-core
  ↓
kaka-core 推送完成通知到 QQ
```

---

## 技术实现

### 1. 数据库层

**新增表：`desktop_operations`**
- 16 个字段，记录操作类型、参数、状态、结果等
- 支持权限分级、决策理由、情绪记录

**Repository：`desktop_repository.py`**
- `create_desktop_operation()` - 创建任务
- `get_pending_operations()` - 获取待执行任务
- `mark_operation_executing()` - 标记执行中
- `complete_operation()` - 完成任务

### 2. API 层

**新增路由：`/v1/desktop`**
- `GET /operations/pending` - 本地组件轮询
- `POST /operations/{id}/start` - 标记开始
- `POST /operations/{id}/complete` - 上传结果并推送通知
- `POST /operations` - 创建任务（内部）
- `GET /operations/{id}` - 查询详情

### 3. 插件系统

**DesktopOperationsPlugin**
- 匹配关键词：在桌面、创建文件、写文件、截图等
- 解析指令：提取文件名、内容
- 权限检查：非创造者不能执行高权限操作
- 立即回复："好的，我来写~"（强调"我"）

### 4. 本地组件

**kaka-local (Python)**

项目结构：
```
apps/kaka-local/
├── src/
│   ├── main.py              # 入口
│   ├── executor.py          # 核心执行器
│   ├── config.py            # 配置管理
│   ├── security.py          # 安全检查
│   └── operations/
│       ├── file_ops.py      # 文件操作
│       ├── screenshot.py    # 截图
│       └── sound.py         # 音效（待实现）
├── .env                     # 配置
└── requirements.txt
```

**安全机制**：
- 路径白名单：只能在 `~/Desktop/卡咔的小角落` 和 `~/Documents/卡咔` 操作
- 文件名检查：禁止路径分隔符、特殊字符
- 扩展名限制：只允许 `.txt`, `.md`, `.json`, `.log`

## 使用示例

### QQ 群触发

```
群友：@卡咔 在桌面写个文件，内容是今天要早睡
卡咔：好的，我来写~
[3秒后]
卡咔：写好了~ 已创建 小纸条.txt
```

实际效果：
- 桌面出现文件夹：`卡咔的小角落/`
- 文件内容：`今天要早睡`

### 支持的指令模式

- "在桌面写个文件，内容是XXX"
- "创建文件，写XXX"
- "桌面写个提醒：XXX"

## 部署说明

### 1. 启动 kaka-core

```bash
cd services/kaka-core
uvicorn kaka_core.api.app:app --host 0.0.0.0 --port 8001
```

### 2. 启动 kaka-local

```bash
cd apps/kaka-local
python src/main.py
```

配置文件 `.env`：
```env
KAKA_CORE_URL=http://127.0.0.1:8001
POLL_INTERVAL_SECONDS=3
```

### 3. 测试

运行集成测试：
```bash
python scripts/test_desktop_integration.py
```

运行演示脚本：
```bash
python scripts/demo_desktop_execution.py
```

## 技术亮点

1. **能力归属明确**：插件话术强调"我来""我写"，不说"助手"
2. **安全可控**：白名单路径、文件名检查、权限分级
3. **架构清晰**：数据库→API→插件→本地组件，各层职责分明
4. **易于扩展**：新增操作类型只需在 operations/ 添加模块
5. **测试覆盖**：集成测试、演示脚本、单元测试全覆盖

## 已知限制

- 本地组件必须运行才能执行任务（不运行则任务堆积在数据库）
