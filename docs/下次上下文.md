# 下一个对话框快速上下文

本文档给新的对话框快速恢复项目进度。建议新对话框优先阅读本文件，再阅读 `KAKA_HANDOFF.md` 和 `docs/开发运行说明.md`。

## 项目一句话

卡咔不是普通 QQ 机器人，而是一个以 `kaka-core` 为核心大脑、未来可接入 QQ/Web/语音/AIoT 的 AI 人格体项目。

核心原则：

```text
卡咔不是 QQ 机器人，QQ 只是卡咔的一个身体。
```

## 用户偏好

- 使用中文交流。
- 用户希望理解每个技术是干什么的，但不希望一开始就被大量复杂代码压住。
- 用户希望开发者负责架构和代码，同时解释“为什么这样做”。
- 用户希望项目保持清晰、可学习、可维护，不要盲目堆功能。
- Python 文件里的注释尽量用中文，关键处适当解释。
- 用户用 PyCharm，喜欢可以直接运行的 `run.py` / `bot.py` 入口。
- 后续新增 Python 注释和 docstring 优先使用中文；技术名词、接口字段、环境变量名和第三方库名称可以保留原文。
- 用户不习惯记命令行参数；开发辅助脚本需要在文件顶部写清楚 PyCharm 右键运行方式、可用参数和常用例子。

```text
inputs：卡咔收到/观察到的输入，包含 analysis_status。
outputs：卡咔对已处理输入形成的输出结果或响应决策，包含 output_origin、output_reason、no_reply_reason；普通观察消息不会写入 outputs。
memory_candidates：待人工审核的长期记忆候选，来源可追溯到 source_input_id。
memories：已经从候选区合并出的正式长期记忆，后续检索只应读取 active 记忆。
auto_job_runs：自动候选分析和自动候选复核的运行记录。
```

状态语义：

```text
inputs.analysis_status:
  not_analyzed：还没有被长期记忆分析处理
  analyzed：已经处理过，或已经产生候选/被人工标记为已处理
  skipped：确认没有长期记忆价值，后续不再分析

memory_candidates.status:
  pending：等待审核或合并
  approved：已经合并进正式 memories
  rejected：确认不进入正式记忆
  merged_duplicate：和已有正式记忆重复，没有新增 memories

memories.status:
  active：参与回复前检索
  archived：保留但不参与回复
```
