## 方案一：本地部署（开发/个人使用）

### 1. 系统要求

- **操作系统**: Windows 10/11, macOS 12+, Ubuntu 20.04+
- **Python**: 3.11+
- **内存**: 至少 2GB
- **硬盘**: 至少 1GB

### 2. 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/your-org/kaka-v2.git
cd kaka-v2

# 2. 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 3. 安装依赖
pip install -r requirements.txt
cd services/kaka-core && pip install -e . && cd ../..
cd apps/qq-adapter && pip install -e . && cd ../..

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 填入真实配置
```

### 3. 启动服务

**终端 1 - kaka-core**:
```bash
cd services/kaka-core
uvicorn kaka_core.api.app:app --host 0.0.0.0 --port 8001
```

**终端 2 - qq-adapter**:
```bash
cd apps/qq-adapter
python -m qq_adapter
```

**终端 3 - kaka-local（可选）**:
```bash
cd apps/kaka-local
python src/main.py
```

### 4. 验证

访问 http://127.0.0.1:8001/docs 查看 API 文档。

## 方案二：Docker 部署

### 1. Dockerfile - kaka-core

```dockerfile
# services/kaka-core/Dockerfile

FROM python:3.11-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制代码
COPY . .
RUN pip install -e .

# 暴露端口
EXPOSE 8001

# 启动命令
CMD ["uvicorn", "kaka_core.api.app:app", "--host", "0.0.0.0", "--port", "8001"]
```

### 2. Dockerfile - qq-adapter

```dockerfile
# apps/qq-adapter/Dockerfile

FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .
RUN pip install -e .

CMD ["python", "-m", "qq_adapter"]
```

### 3. docker-compose.yml

```yaml
version: '3.8'

services:
  kaka-core:
    build:
      context: ./services/kaka-core
      dockerfile: Dockerfile
    ports:
      - "8001:8001"
    environment:
      - LLM_ENABLED=true
      - LLM_API_KEY=${LLM_API_KEY}
      - LLM_BASE_URL=${LLM_BASE_URL}
    volumes:
      - ./data:/app/data
    restart: unless-stopped

  qq-adapter:
    build:
      context: ./apps/qq-adapter
      dockerfile: Dockerfile
    ports:
      - "8081:8081"
    environment:
      - KAKA_CORE_BASE_URL=http://kaka-core:8001
      - ONEBOT_WS_URL=${ONEBOT_WS_URL}
    depends_on:
      - kaka-core
    restart: unless-stopped

  kaka-local:
    build:
      context: ./apps/kaka-local
      dockerfile: Dockerfile
    environment:
      - KAKA_CORE_URL=http://kaka-core:8001
    volumes:
      - ${HOME}/.kaka:/root/.kaka
    depends_on:
      - kaka-core
    restart: unless-stopped
```

### 4. 启动

```bash
# 创建 .env 文件
cp .env.example .env

# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 方案三：Nginx 反向代理

### 1. Nginx 配置

```nginx
# /etc/nginx/sites-available/kaka

server {
    listen 80;
    server_name kaka.example.com;

    # kaka-core API
    location /api/ {
        proxy_pass http://127.0.0.1:8001/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 管理后台
    location /admin/ {
        proxy_pass http://127.0.0.1:8001/admin/;
        # 可选：Basic Auth
        auth_basic "Kaka Admin";
        auth_basic_user_file /etc/nginx/.htpasswd;
    }

    # 静态文件（如果有）
    location /static/ {
        alias /var/www/kaka/static/;
        expires 30d;
    }
}
```

### 2. HTTPS (Let's Encrypt)

```bash
# 安装 certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d kaka.example.com

# 自动续期
sudo certbot renew --dry-run
```

## 方案四：Systemd 服务

### 1. kaka-core.service

```ini
# /etc/systemd/system/kaka-core.service

[Unit]
Description=Kaka Core Service
After=network.target

[Service]
Type=simple
User=kaka
WorkingDirectory=/opt/kaka/services/kaka-core
Environment="PATH=/opt/kaka/.venv/bin"
ExecStart=/opt/kaka/.venv/bin/uvicorn kaka_core.api.app:app --host 0.0.0.0 --port 8001
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 2. qq-adapter.service

```ini
# /etc/systemd/system/qq-adapter.service

[Unit]
Description=Kaka QQ Adapter
After=network.target kaka-core.service
Requires=kaka-core.service

[Service]
Type=simple
User=kaka
WorkingDirectory=/opt/kaka/apps/qq-adapter
Environment="PATH=/opt/kaka/.venv/bin"
ExecStart=/opt/kaka/.venv/bin/python -m qq_adapter
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 3. 管理服务

```bash
# 启动服务
sudo systemctl start kaka-core
sudo systemctl start qq-adapter

# 开机自启
sudo systemctl enable kaka-core
sudo systemctl enable qq-adapter

# 查看状态
sudo systemctl status kaka-core

# 查看日志
sudo journalctl -u kaka-core -f
```

## 数据备份

### 1. 备份脚本

```bash
#!/bin/bash
# scripts/backup.sh

BACKUP_DIR="/backup/kaka"
DATE=$(date +%Y%m%d_%H%M%S)

# 备份数据库
cp data/kaka.sqlite3 "$BACKUP_DIR/kaka_$DATE.sqlite3"

# 备份配置
cp .env "$BACKUP_DIR/env_$DATE"

# 压缩旧备份（保留最近 7 天）
find "$BACKUP_DIR" -name "*.sqlite3" -mtime +7 -delete

echo "Backup completed: $BACKUP_DIR/kaka_$DATE.sqlite3"
```

### 2. 定时备份 (cron)

```bash
# 每天凌晨 2 点备份
0 2 * * * /opt/kaka/scripts/backup.sh >> /var/log/kaka-backup.log 2>&1
```

## 监控与日志

### 1. 日志配置

```python
# services/kaka-core/src/kaka_core/config/logging.py

import logging.config

LOGGING_CONFIG = {
    "version": 1,
    "disable_existing_loggers": False,
    "formatters": {
        "default": {
            "format": "%(asctime)s - %(name)s - %(levelname)s - %(message)s",
        },
    },
    "handlers": {
        "console": {
            "class": "logging.StreamHandler",
            "formatter": "default",
        },
        "file": {
            "class": "logging.handlers.RotatingFileHandler",
            "filename": "/var/log/kaka/kaka-core.log",

            "backupCount": 5,
            "formatter": "default",
        },

    "root": {
        "level": "INFO",
```

## 安全建议

1. **使用 HTTPS**: 生产环境必须启用 HTTPS
2. **环境变量加密**: 不要在代码中硬编码密钥
3. **API 鉴权**: 为敏感 API 添加 Token 验证
4. **防火墙**: 限制端口访问
5. **定期更新**: 及时更新依赖库
6. **备份**: 定期备份数据库
7. **日志脱敏**: 不记录敏感信息

## 故障排查

### 问题 1: kaka-core 无法启动

```bash
# 检查端口占用
lsof -i :8001

# 查看日志
tail -f /var/log/kaka/kaka-core.log

# 检查依赖
pip list | grep fastapi
```

### 问题 2: QQ 适配器连接失败

```bash
# 检查 NapCat 是否运行
curl http://127.0.0.1:3000/

# 检查 WebSocket 连接
python -c "import websockets; print('WebSocket available')"

# 查看适配器日志
journalctl -u qq-adapter -f
```

### 问题 3: 数据库锁定

```bash
# 检查数据库文件权限
ls -la data/kaka.sqlite3

# 检查是否有进程占用
lsof data/kaka.sqlite3

# 重启服务
sudo systemctl restart kaka-core
```
