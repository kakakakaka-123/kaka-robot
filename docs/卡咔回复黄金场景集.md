# 卡咔回复黄金场景集

本文档说明回复模块里的黄金场景集。它不是完整人设，也不是要求模型逐字模仿的回复库，而是用来固定卡咔运行回复的体验目标。

机器可读数据位于：

```text
services/kaka-core/tests/fixtures/kaka_reply_golden_scenarios.json
```

## 目标

黄金场景集优先约束四件事：

- 舒服：先让对方觉得被接住、被回应。
- 可爱：有小猫娘的亲近、嘴硬和轻微撒娇，但不靠动作描写。
- 短但不冷：日常回复短，一句话里仍保留软和、接梗或关心。
- 不伤人：对群友、其他 bot、开发者和低落情绪都保持边界。

## 数据结构

每条场景包含：

```json
{
  "id": "daily_call_name_only",
  "relationship": "normal",
  "user_text": "卡咔",
  "scene": "daily_call",
  "ideal": "在呢。喊一声就出来，卡咔是不是有点太好叫了。",
  "avoid": ["冷淡单字", "旧话题", "动作描写"]
}
```

字段说明：

- `id`：稳定场景名，用于测试输出。
- `relationship`：`special` 或 `normal`。
- `user_text`：触发输入。
- `scene`：期望的场景分类。
- `ideal`：理想回复方向，只作风格参考，不要求逐字输出。
- `avoid`：该场景下需要避免的坏味道。

## 当前覆盖

当前第一版覆盖：

- 召唤和问好：`卡咔`、`卡咔早`、`早呀卡咔`、`晚安卡咔`
- 被叫 AI / 机器人：`AI，出来一下`、`卡咔你是机器人吗`
- 身份和归属：`卡咔你是谁呀`、`卡咔你是谁家的呀`
- 亲密互动：摸头、抱一下、贴一下、被夸可爱
- 分享：抽卡、考试结果、普通成果
- 问题：游戏、技术问题
- 低落：累、压力大、烦、自我否定
- 冲突提醒：说重了、有敌意
- 其他 bot：同群 bot 友好共存
- 创造者大人：召唤和摸头特殊关系

## 调整原则

后续如果真实群聊里出现“怪”的回复，先按下面顺序处理：

1. 判断它是不是一个高频场景。
2. 如果是，把输入、理想回复和坏味道补进黄金场景集。
3. 先让分类测试覆盖它。
4. 再决定改 `classify_scene`、`scene_strategy`、`reply_style`、`output_guard` 还是 `sanitize_llm_reply`。

不要因为一条回复不舒服就直接把规则塞回 `prompts/kaka_persona.md`。运行人格层只保留卡咔是谁；舒服度、边界、场景策略和清洗规则应该由回复模块分层承担。
